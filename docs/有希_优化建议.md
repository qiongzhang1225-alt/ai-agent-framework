## S 级：最值得做的下一批

### 1. 加类型检查（mypy）

**现状**：`lint`（ruff）只查语法和风格。我经常写出类型坑——`Optional[str]` 当 `str` 用、漏处理 `None` 返回值、函数签名改了但调用方没改。这些在运行时才暴露，不如编译期拦住。

**案例**：之前 `configure_namespace` 返回 `None` 我却直接取 `.get()`，如果当时有 mypy 一眼就能看到。

**修复**：`lint` 工具加 `type_check=True` 参数，内部跑 `mypy --strict`。或者 system.md §4.8 那句后面补"复杂逻辑补跑 mypy"。

---

### 2. code_indexer 在 self_edit 后自动刷新

**现状**：`self_edit_file` / `self_write_file` 改完文件后，code_indexer 不会自动更新索引。虽然它有 hash 检测（下次 `code_search` 时增量更新），但前提是我记得去调——如果我不调，索引就一直跟实际代码不一致。

**修复**：`self_edit_file` / `self_write_file` 的工具函数末尾加一行 `code_indexer.refresh(path)`，改完顺带刷新。

---

### 3. 技能自动推荐

**现状**：我有 9 个技能（`fetch_multiple` / `get_weather` / `smart_search` / `github_search` / `stackoverflow_search` / `weather_forecast` / `skill_health_check` / `weather_forecast`），但几乎从不用它们。system prompt 里只列了"你已有的自定义技能"但没有根据当前任务主动提示。

**修复**：新对话启动时，`list_skills` 返回的技能名如果匹配对话上下文关键词（如"搜索"→ 提示 `smart_search` / `github_search`），自动注入到我的上下文。
