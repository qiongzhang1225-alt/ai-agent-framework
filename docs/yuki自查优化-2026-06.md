## 实现要点

### code_indexer.update_file 与 refresh 的区别

yuki 自查时建议用 `refresh()`，但看代码：

```python
def refresh(self, path: str) -> None:
    self._caches.pop(path, None)       # 把整个 root 缓存清掉
    self._ensure_index(path, "python") # 走全量索引分支（200+ 文件）
```

每次 `self_edit` 都触发**全量重建**是浪费。新加的 `update_file(root, relpath)` 只重新索引一个文件，配合 `_index_file()` 内部已有的"先清掉该文件老数据再 walk"逻辑，干净且快。

### lint type_check 为什么默认关

- `mypy --strict` 在缺类型注解的项目里会爆出几百条噪音
- 非 strict 也漏检不少类型坑（duck typing 容忍度太高）
- 不适合作"改完 .py 必跑"硬约束 —— 误报会让 yuki 反复修无意义的东西
- 改成"按需触发"：改了涉及类型的关键代码 + 主人说"查下类型" 时主动开

### sub_completion 同时进 conv.json + 长期记忆

子对话结束摘要既写 `conv.json` 的 `completion_summary` 字段（结构化），又通过 `add_memory(category="sub_completion", importance=7)` 进 ChromaDB（语义可检索）。master 下次启动时 `format_recent_sub_completions_for_prompt()` 注入最近 5 条到 SYSTEM_PROMPT，跨对话上下文延续。
